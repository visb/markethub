# S0.5 — Pacotes compartilhados (types, api-client, ui)

- **Fase:** 0
- **Epic:** Fundação
- **Status:** done
- **Depende de:** [S0.1, S0.4]

## Objetivo
Pacotes reutilizáveis para contratos, cliente de API e UI, eliminando duplicação entre apps.

## User story
Como dev, quero tipos e cliente de API compartilhados, para frontend e backend ficarem sempre em sincronia.

## Critérios de aceite
- [ ] `packages/types`: schemas zod + tipos inferidos (auth, user; catálogo/pedido entram depois). Fonte única de verdade dos DTOs.
- [ ] `packages/api-client`: cliente HTTP tipado (fetch/axios) consumindo `types`, com injeção de token e refresh automático; stub de cliente Socket.IO.
- [ ] `packages/ui`: design tokens (cores, espaçamento, tipografia derivados dos screenshots) + 2-3 componentes RN base (Button, Text, Screen).
- [ ] Pacotes buildáveis e importáveis por `apps/*` via workspace.
- [ ] Versionamento interno via `workspace:*`.

## Escopo / Fora de escopo
Inclui: scaffolding e contratos iniciais dos 3 pacotes.
Não inclui: biblioteca de componentes completa, theming avançado.

## Notas técnicas
- `types` não importa nada de runtime de Nest/RN — pacote puro.
- `api-client` reusa o formato de erro definido na S0.2.
- `ui` compartilhado entre customer/picker/driver; admin (web) pode ter UI própria.
