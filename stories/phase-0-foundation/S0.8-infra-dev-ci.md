# S0.8 — Infra dev & CI

- **Fase:** 0
- **Epic:** Fundação
- **Status:** done
- **Depende de:** [S0.2, S0.3]

## Objetivo
Ambiente local reprodutível (Postgres/Redis/MinIO) e pipeline de CI validando o monorepo.

## User story
Como dev, quero subir as dependências com um comando e ter CI verde, para desenvolver e integrar com confiança.

## Critérios de aceite
- [ ] `infra/docker-compose.yml` com Postgres, Redis e MinIO (S3-compatível).
- [ ] `.env.example` raiz documentando todas as variáveis (DB, Redis, S3, JWT, gateways).
- [ ] Script `pnpm infra:up` / `infra:down` para subir/derrubar os serviços.
- [ ] CI (GitHub Actions) rodando em PR: install → lint → typecheck → build → test.
- [ ] CI sobe serviços necessários (Postgres) para testes que tocam o banco.
- [ ] README com passo a passo de setup do zero.

## Escopo / Fora de escopo
Inclui: compose de dev, env, CI de verificação.
Não inclui: CD/deploy, infra de produção (IaC), observabilidade prod.

## Notas técnicas
- Redis e MinIO já provisionados aqui, mesmo que só sejam usados na Fase 1+ (filas/enriquecimento/imagens).
- Cache do Turbo no CI para acelerar builds.
