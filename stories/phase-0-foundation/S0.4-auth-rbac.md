# S0.4 — Auth & RBAC

- **Fase:** 0
- **Epic:** Fundação
- **Status:** done
- **Depende de:** [S0.3]

## Objetivo
Autenticação por JWT com refresh e autorização por papéis, usada por todos os apps.

## User story
Como usuário (cliente/separador/entregador/admin), quero logar com segurança, para acessar só o que meu papel permite.

## Critérios de aceite
- [ ] Endpoints: `POST /auth/register`, `POST /auth/login`, `POST /auth/refresh`, `POST /auth/logout`.
- [ ] Senhas com hash (argon2/bcrypt). Nunca retornar hash.
- [ ] Access token (curto) + refresh token (longo, persistido em `Session`, rotacionável/revogável).
- [ ] `JwtAuthGuard` global + decorator `@Public()` para rotas abertas.
- [ ] `RolesGuard` + decorator `@Roles('admin', ...)` para autorização por papel.
- [ ] `GET /auth/me` retorna perfil + papéis do usuário autenticado.
- [ ] Testes cobrindo login ok, senha errada, refresh rotacionado, acesso negado por papel.

## Escopo / Fora de escopo
Inclui: fluxo de credenciais, tokens, guards, RBAC.
Não inclui: OAuth social, OTP/SMS, recuperação de senha (backlog).

## Notas técnicas
- Contratos de auth em `packages/types` (reusados pelo `api-client` e apps).
- Refresh token: detectar reuse (token rotation) e revogar a sessão.
- Segredos via env (S0.2).
