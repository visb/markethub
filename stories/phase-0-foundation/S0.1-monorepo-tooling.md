# S0.1 — Monorepo + tooling

- **Fase:** 0
- **Epic:** Fundação
- **Status:** done
- **Depende de:** —

## Objetivo
Esqueleto do monorepo pronto para receber os apps e pacotes, com build/lint/format unificados.

## User story
Como dev, quero um monorepo configurado, para adicionar apps e compartilhar código sem retrabalho.

## Critérios de aceite
- [ ] Turborepo + pnpm workspaces configurados (`pnpm-workspace.yaml`, `turbo.json`).
- [ ] Estrutura `apps/` e `packages/` criada (vazias com `.gitkeep` ou READMEs).
- [ ] TS base compartilhado (`tsconfig.base.json`) estendido pelos pacotes.
- [ ] ESLint + Prettier na raiz, com scripts `lint` e `format`.
- [ ] Scripts raiz: `pnpm dev`, `pnpm build`, `pnpm lint`, `pnpm test` orquestrados pelo Turbo.
- [ ] `.gitignore`, `.editorconfig`, `README.md` raiz com instruções de setup.
- [ ] Git inicializado (repo ainda não é git — `git init`).

## Escopo / Fora de escopo
Inclui: configuração de monorepo, tooling, convenções.
Não inclui: código de app/backend, CI (fica na S0.8).

## Notas técnicas
- `package.json` raiz com `packageManager` pinado.
- Turbo pipeline: `build` depende de `^build`; `dev` sem cache.
- Node LTS; pnpm via corepack.
