# S4.2 Disponibilidade e localização do entregador
- **Fase:** 4
- **Epic:** Disponibilidade
- **Status:** todo
- **Depende de:** [S4.1]

## Objetivo
Permitir ao entregador ficar disponível/indisponível e reportar localização, base para o matching (ref: `Home - Pressing Status toggle.jpg`).

## User story
Como entregador, quero alternar meu status e enviar minha posição, para receber ofertas de rota quando estiver disponível.

## Critérios de aceite
- `PATCH /driver/status` com `status` (`offline | available`) — só `driver`
- `available` exige localização (`lat`/`lng`); atualiza `lastSeenAt`
- `POST /driver/location` (heartbeat) atualiza `currentLat`/`currentLng`/`lastSeenAt`
- Entregador em rota (`on_route`) não pode ficar `offline` até concluir/cancelar
- `GET /driver/me` retorna perfil + status + rota ativa (se houver)
- Só entregadores `available` (e com `lastSeenAt` recente) entram no pool de matching (S4.3)

## Escopo / Fora de escopo
- **Fora:** stream de localização ao cliente (Fase 5)
- **Inclui:** toggle de status, heartbeat de posição, leitura do próprio estado

## Notas técnicas
- `lastSeenAt` com janela de staleness (ex: 2 min) para considerar online
- Transições de status validadas (não ir a `offline` com rota ativa)
- Localização em `Float` lat/lng; índice por `status` para o pool
