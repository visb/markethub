# S5.8 Alinhamento do app cliente com as telas de referência (marketplace)
- **Fase:** 5
- **Epic:** Polish
- **Status:** done
- **Depende de:** S5.1, S5.2, S5.3, S3.4

## Objetivo
Ajustar o app `customer` ao resultado esperado em `briefing/screenshots/marketplace/*` e
fechar lacunas de workflow nos demais apps (picker) para que o fluxo ponta a ponta
(comprar → pagar → acompanhar → substituições → receber → avaliar) funcione como nas telas.

## User story
Como cliente, acompanho meu pedido com previsão de entrega, vejo o progresso da separação
item a item, decido substituições propostas pelo separador, recebo com senha e avalio cada
mercado do pedido — como nas telas de referência.

## Critérios de aceite
- [x] Acompanhe seu pedido (ref `Confirmed.jpg`): previsão de entrega (janela agendada ou
      estimada), endereço, 3 macro-etapas (confirmado/comprando/a caminho), detalhes por
      mercado com subtotal e total, cancelar pedido.
- [x] Comprando (ref `Picking.jpg`): contadores "a escolher / selecionados / reembolsados /
      a selecionar" e aprovação/recusa de substituições inline (API S3.4 já existia; UI nova).
- [x] Cancelamento ampliado: `created|paid|preparing` enquanto nenhuma separação começou;
      pedido pago gera estorno integral (`Refund` reason `customer_cancel`), tarefas saem da fila.
- [x] Seu pedido chegou (ref `Receive.jpg`): senha de entrega/retirada em destaque quando
      a caminho / pronto p/ retirar.
- [x] PIX (ref `Payment.jpg`): QR code real do payload copia-e-cola + copiar para a área de
      transferência (`react-native-qrcode-svg`, `expo-clipboard`).
- [x] Tudo certo! (ref `Order Completed.jpg`): check verde, eixos MarketHub/Entrega/cada
      mercado (review por merchant: unique `orderId+axis+targetMerchantId`, migração
      `20260611000000_review_per_merchant`), gorjeta com checkbox+stepper, comentário único,
      Concluir envia tudo.
- [x] Minhas compras (ref `Orders.jpg`): "Entrega prevista para"/"Entregue em" + total à direita.
- [x] Carrinho (ref `Cart.jpg`): taxa de entrega real por mercado (`totals.groups`) e cupom
      funcional (aplicar/remover).
- [x] Finalizar compra (ref `Shipping Settings.jpg`): valor da entrega + desconto do cupom
      no card de entrega.
- [x] Home (ref `Home.png`): atalhos flutuantes acima do carrinho para as lojas que já têm
      itens no carrinho (aparecem conforme produtos de mercados diferentes são adicionados;
      sem logo no modelo → círculo com iniciais do mercado).
- [x] Picker (complemento): propor substituto buscando ofertas da loja (substitui
      `Alert.prompt`, que não existe no Android).
- [x] Tracking API: `etaWindow`, `address`, `totalCents`, por grupo `merchantId/merchantName/
      subtotalCents/picking{toApprove,picked,refused,pending}`.
