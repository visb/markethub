# S5.4 Dashboard admin (pedidos, operações, financeiro)
- **Fase:** 5
- **Epic:** Admin
- **Status:** todo
- **Depende de:** [S3.11, S4.7, SF.3]

## Objetivo
Painel admin (Vite SPA) com visão operacional e financeira: pedidos por status, separação/rotas em andamento e financeiro (vendas, reembolsos, gorjetas, repasses).

## User story
Como admin, quero acompanhar pedidos, operação e financeiro num painel, para gerir a plataforma.

## Critérios de aceite
- **Pedidos:** lista/busca por status, loja e período; detalhe com grupos, separação, rota e pagamento/reembolso.
- **Operações:** separações ativas (filas por loja) e rotas (`offered/accepted/in_progress/completed`), com SLA básico.
- **Financeiro:** agregados por período — vendas (`Payment.paid`), reembolsos (`Refund`, SF.3), gorjetas (`Tip`, S5.2) e repasse estimado ao merchant/entregador.
- Endpoints admin agregados (somente `admin`); páginas Vite reusando o shell de S0.7/S3.11.

## Escopo / Fora de escopo
- **Inclui:** endpoints de agregação, páginas de pedidos/operações/financeiro.
- **Fora:** exportações fiscais; conciliação bancária; gráficos avançados.

## Notas técnicas
- Stack admin é **Vite + React SPA** (não Next) — manter o padrão existente.
- Reusa `Refund`/`RefundComponent` p/ o recorte de reembolsos por grupo.
