# S3.10 Cadastro de produto canônico pelo merchant (API)
- **Fase:** 3
- **Epic:** Gestão merchant (catálogo/estoque)
- **Status:** todo
- **Depende de:** [S1.1, S1.5, S3.9]

## Objetivo
Permitir ao manager criar/editar produto canônico (`Product`) para itens sem GTIN/ERP, criando a `Offer` na sua loja.

## User story
Como manager, quero cadastrar um produto que não vem do ERP (ex: item de produção própria), para vendê-lo na minha loja.

## Critérios de aceite
- `POST /merchant/products` — cria `Product` (nome, marca, `saleType`, `packageSize`, imagem, categoria) + `Offer` na loja do manager
- `PATCH /merchant/products/:id` — editar campos do `Product`; campos editados entram em `lockedFields` (não sobrescritos pelo enrichment S1.5)
- **Guard de duplicidade:** se informar GTIN, checar `Product.gtin` único; se já existe, propor reuso (criar só `Offer`) em vez de duplicar canônico
- Sem GTIN: produto é "local" do merchant; busca de similares (nome/marca) alerta possível duplicata
- Manager só edita `Product` que ele criou ou que só tem `Offer`(s) das lojas dele (não edita canônico compartilhado entre merchants)
- Upload de imagem (storage S3/MinIO)

## Escopo / Fora de escopo
- **Fora:** edição de canônico compartilhado (fica com admin/ERP), UI (S3.11)
- **Inclui:** criação/edição de Product local + Offer, dedup guard, upload

## Notas técnicas
- Marcar origem do `Product` (ex: `source=merchant`) p/ distinguir de ERP/Cosmos
- Enrichment (S1.5) pode rodar p/ produto com GTIN, respeitando `lockedFields`
- Reusar validações/serviço de catálogo do S1.7 (escopados)
