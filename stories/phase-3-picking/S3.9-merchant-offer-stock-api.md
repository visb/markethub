# S3.9 Gestão de ofertas e estoque pelo merchant (API)
- **Fase:** 3
- **Epic:** Gestão merchant (catálogo/estoque)
- **Status:** done
- **Depende de:** [S1.1, S1.4, S0.4]

## Objetivo
API escopada para o manager do merchant gerir preço, disponibilidade e estoque das próprias lojas.

## User story
Como manager de um merchant, quero ajustar preço, promoção e estoque dos meus produtos, sem depender do admin da plataforma.

## Critérios de aceite
- `GET /merchant/offers` — `Offer`s das lojas do manager (filtro por loja, categoria, busca, disponibilidade)
- `PATCH /merchant/offers/:id` — `priceCents`, `promoPriceCents`, `available`
- `GET /merchant/stocks` / `PATCH /merchant/stocks/:id` — `quantity`, `available`
- Escopo obrigatório: manager só toca `Offer`/`Stock` de `Store` onde tem `StoreStaff staffRole=manager` (RBAC `merchant`/`manager` + guard de `storeId`)
- **Lock por campo:** campo editado manualmente entra em `lockedFields`; sync ERP (S1.4) não sobrescreve campo locked
- Destravar campo (remover do lock) volta o controle ao ERP
- Auditoria: autor + timestamp por alteração

## Escopo / Fora de escopo
- **Fora:** criar/editar produto canônico (S3.10), UI (S3.11)
- **Inclui:** endpoints de Offer/Stock escopados + mecanismo de lock
