# S3.11 Admin SPA: área do merchant (manager)
- **Fase:** 3
- **Epic:** Gestão merchant (catálogo/estoque)
- **Status:** done
- **Depende de:** [S0.7, S1.7, S3.9, S3.10]

## Objetivo
Estender a admin SPA (Vite + React) com área escopada para o manager gerir catálogo e estoque da sua loja.

## User story
Como manager, quero uma tela web para editar preços, estoque e produtos da minha loja, reaproveitando o painel existente.

## Critérios de aceite
- Login `merchant`/`manager` na admin SPA; navegação e telas filtradas por papel + loja
- Manager vê só dados das suas lojas (sem acesso às telas globais de admin)
- Lista de ofertas: editar preço/promo/disponibilidade inline (S3.9)
- Tela de estoque: ajustar quantidade/disponibilidade (S3.9)
- Cadastro/edição de produto local + upload de imagem (S3.10)
- Indicador visual de campo "locked" (editado manual, ignora ERP) + ação destravar
- Seletor de loja quando o manager gere mais de uma

## Escopo / Fora de escopo
- **Fora:** telas de admin global (S1.7 permanecem só p/ admin)
- **Inclui:** área manager na admin SPA reusando componentes de catálogo

## Notas técnicas
- Reusar componentes/serviços de S1.7; gating por papel (`admin` vs `merchant`/`manager`)
- Consumir endpoints `/merchant/*` (S3.9/S3.10) via api-client
- Estados loading/erro/vazio; otimista onde fizer sentido
