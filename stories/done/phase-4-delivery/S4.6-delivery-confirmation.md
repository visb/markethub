# S4.6 Entrega: chegada e confirmação por código
- **Fase:** 4
- **Epic:** Entrega
- **Status:** superseded — entrega agora por Delivery (driverConfirmDelivery + deliveryCode), sem RouteStop. Retirada via handover. Ver delivery-own-store-model.
- **Depende de:** [S4.5]

## Objetivo
Executar a parada de entrega: chegar ao cliente e confirmar a entrega digitando o `deliveryCode` informado pelo cliente (ref: `Entrega.jpg`, `Entrega - Chegou.jpg`).

## User story
Como entregador, quero confirmar a entrega digitando o código que o cliente me mostra, para comprovar a entrega sem depender de QR/senha de caixa.

## Critérios de aceite
- `POST /driver/routes/:id/stops/:stopId/arrive` → `arrived` (header "Entrega (1/N)" — ref screenshot)
- `POST /driver/routes/:id/stops/:stopId/confirm` confirma a entrega validando o `deliveryCode` (do `Order`, exibido ao cliente — SF.1)
- Ao confirmar → todos os `OrderGroup` do `Order` na parada viram `delivered`; quando todos os grupos do `Order` ficam `delivered` → `Order → delivered`
- `POST /driver/routes/:id/stops/:stopId/complete` ("Finalizar Entrega") fecha a parada (exige entrega confirmada)
- Código inválido → erro (`INVALID_DELIVERY_CODE`); sem marcar entregue

## Escopo / Fora de escopo
- **Fora:** avaliação/gorjeta do cliente (Fase 5)
- **Inclui:** chegada, confirmação por `deliveryCode`, transição para `delivered`

## Notas técnicas
- `deliveryCode` gerado na criação do pedido (SF.1); recompõe status agregado do `Order` (como em S3.6)
- Idempotência por `Order`; foto/assinatura opcional fica fora
- Rate-limit / limite de tentativas na validação do código (anti-brute-force)
- Conclusão da última parada → `DeliveryRoute → completed` (gatilho de S4.7)
