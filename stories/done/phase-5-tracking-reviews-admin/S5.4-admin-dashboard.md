# S5.4 Dashboard admin (pedidos, operações, financeiro)
- **Fase:** 5
- **Epic:** Admin
- **Status:** done
- **Depende de:** [S3.11, SF.3, entrega própria (Delivery)]

## Objetivo
Painel admin (Vite SPA) com visão operacional e financeira: pedidos por status, separação/entregas em andamento e financeiro (vendas, reembolsos, gorjetas).

## User story
Como admin, quero acompanhar pedidos, operação e financeiro num painel, para gerir a plataforma.

## Critérios de aceite
- **Pedidos:** lista/busca por status, loja e período; detalhe com grupos, separação, entrega (`Delivery`) e pagamento/reembolso.
- **Operações:** separações ativas (filas por loja) e entregas por loja (`unassigned/assigned/picked_up/delivered`) + retiradas pendentes, com SLA básico.
- **Financeiro:** agregados por período — vendas (`Payment.paid`), reembolsos (`Refund`, SF.3), gorjetas (`Tip`, S5.2) e repasse estimado ao merchant. Sem repasse/ganhos por rota ao entregador (entregador é staff da loja).
- Endpoints admin agregados (somente `admin`); páginas Vite reusando o shell de S0.7/S3.11.

## Escopo / Fora de escopo
- **Inclui:** endpoints de agregação, páginas de pedidos/operações (separação + entregas)/financeiro.
- **Fora:** exportações fiscais; conciliação bancária; gráficos avançados.

## Notas técnicas
- Stack admin é **Vite + React SPA** (não Next) — manter o padrão existente.
- Reusa `Refund`/`RefundComponent` p/ o recorte de reembolsos por grupo.
