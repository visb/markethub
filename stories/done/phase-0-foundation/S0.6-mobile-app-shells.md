# S0.6 — Shells dos apps mobile (Expo)

- **Fase:** 0
- **Epic:** Fundação
- **Status:** done
- **Depende de:** [S0.5]

## Objetivo
Três apps Expo (customer, picker, driver) inicializados, com navegação e login ligados à API.

## User story
Como usuário de cada app, quero abrir o app e logar, para acessar a área autenticada do meu papel.

## Critérios de aceite
- [ ] `apps/customer`, `apps/picker`, `apps/driver` criados com Expo + TypeScript.
- [ ] Navegação configurada (expo-router ou react-navigation) com stacks: Auth → App.
- [ ] Tela de login funcional usando `packages/api-client` (login/refresh reais contra a API).
- [ ] Sessão persistida (secure store) e auto-login se token válido.
- [ ] Cada app valida o papel correto no login (ex.: driver exige papel `driver`).
- [ ] Home placeholder por app exibindo nome do usuário logado (`/auth/me`).
- [ ] Usam `packages/ui` para componentes base.

## Escopo / Fora de escopo
Inclui: bootstrap dos 3 apps, navegação, auth, telas placeholder.
Não inclui: telas de negócio (Fases 2-4), push, deep links.

## Notas técnicas
- Compartilhar config de tema via `packages/ui`.
- Apontar para API local via env do Expo.
- Estrutura de pastas idêntica entre os 3 apps para facilitar manutenção.
