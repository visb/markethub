# S0.3 — DB + Prisma

- **Fase:** 0
- **Epic:** Fundação
- **Status:** done
- **Depende de:** [S0.2]

## Objetivo
PostgreSQL + Prisma integrados, com schema base e migrations versionadas.

## User story
Como dev, quero persistência configurada, para modelar e gravar dados de domínio.

## Critérios de aceite
- [ ] Prisma instalado em `apps/api` (ou `packages/db` se preferir compartilhar client).
- [ ] `schema.prisma` com datasource Postgres + generator.
- [ ] Modelos base: `User`, `Role`, `Session` (refresh tokens), enum de papéis.
- [ ] Primeira migration criada e aplicada (`prisma migrate dev`).
- [ ] `PrismaModule`/`PrismaService` injetável no Nest, com shutdown hooks.
- [ ] Seed script (`prisma db seed`) cria 1 usuário admin de exemplo.
- [ ] `/health` passa a checar conexão com o banco.

## Escopo / Fora de escopo
Inclui: setup Prisma, schema de identidade, migrations, seed mínimo.
Não inclui: modelos de catálogo/pedido (Fase 1+), lógica de auth (S0.4).

## Notas técnicas
- Papéis: `customer`, `picker`, `driver`, `merchant`, `admin`.
- `User` pode ter múltiplos papéis (relação N:N) para suportar testes (um humano = vários apps).
- Postgres provido pelo docker-compose da S0.8 (em dev pode ser container avulso até lá).
