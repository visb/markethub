# S0.7 — Shell admin web (Vite + React)

- **Fase:** 0
- **Epic:** Fundação
- **Status:** done
- **Depende de:** [S0.5]

## Objetivo
Painel administrativo SPA (Vite + React) com login e layout base, pronto para receber telas de gestão.

## User story
Como admin, quero acessar um painel web e logar, para gerenciar catálogo, pedidos e operações.

## Critérios de aceite
- [x] `apps/admin` com Vite + React + TypeScript (SPA, sem Next.js).
- [x] Login usando `packages/api-client`, exigindo papel `admin`.
- [x] Sessão protegida (`ProtectedRoute` redireciona não autenticados para /login — equivalente SPA do middleware).
- [x] Layout base: sidebar + topbar + área de conteúdo (navegação placeholder p/ Catálogo, Pedidos, ERP).
- [x] Página inicial pós-login com dados do admin (`/auth/me`).
- [x] Estilos próprios (CSS) com tokens de marca; shadcn/Tailwind opcional depois.

## Escopo / Fora de escopo
Inclui: scaffolding, auth, layout, navegação vazia.
Não inclui: telas de catálogo/pedido/ERP (Fases 1+).

## Notas técnicas
- Decisão: **Vite + React SPA** no lugar de Next.js (escolha do usuário).
- Roteamento `react-router-dom` v6; proteção client-side via `ProtectedRoute`.
- TokenStore em `localStorage` (`LocalTokenStore`).
- Vite resolve `@markethub/api-client` e `@markethub/types` direto do source TS (alias no `vite.config.ts`)
  para evitar interop CJS/ESM do dist.
- Roda em `localhost:3001` (`pnpm --filter @markethub/admin dev`).
