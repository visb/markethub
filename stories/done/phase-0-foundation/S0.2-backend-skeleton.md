# S0.2 — Backend skeleton (NestJS)

- **Fase:** 0
- **Epic:** Fundação
- **Status:** done
- **Depende de:** [S0.1]

## Objetivo
API NestJS rodando, com config por ambiente, health-check, tratamento de erro e logging.

## User story
Como dev, quero um backend base padronizado, para construir módulos de domínio sobre ele.

## Critérios de aceite
- [ ] `apps/api` com NestJS, estrutura modular (módulos por domínio virão depois).
- [ ] `ConfigModule` com validação de env (zod ou class-validator) e `.env.example`.
- [ ] Endpoint `GET /health` retornando status do processo (e dependências quando existirem).
- [ ] Filtro global de exceções → formato de erro JSON consistente (`{ code, message, details }`).
- [ ] Interceptor de logging (request id, latência) com logger estruturado (pino).
- [ ] CORS e validação global de payload (ValidationPipe) ligados.
- [ ] `pnpm --filter api dev` sobe a API; `pnpm --filter api build` compila.

## Escopo / Fora de escopo
Inclui: bootstrap, config, observabilidade básica, contrato de erro.
Não inclui: Prisma (S0.3), auth (S0.4), módulos de negócio.

## Notas técnicas
- Versionamento de API: prefixo `/api/v1`.
- Padrão de erro reutilizado pelo `packages/api-client`.
