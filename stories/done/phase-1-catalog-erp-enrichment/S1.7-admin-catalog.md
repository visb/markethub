# S1.7 — Admin de catálogo

- **Fase:** 1
- **Epic:** Catálogo / Enriquecimento
- **Status:** done
- **Depende de:** [S0.7, S1.5, S1.6]

## Objetivo
Telas de administração para inspecionar o catálogo, ver status de enriquecimento e corrigir manualmente.

## User story
Como admin, quero ver e ajustar produtos, para garantir qualidade do catálogo e corrigir o que o pipeline errou.

## Critérios de aceite
- [ ] Listagem de produtos com busca/filtro (por merchant, categoria, score de completude, status).
- [ ] Detalhe do produto: dados canônicos, ofertas por loja, origem de cada campo (provenance).
- [ ] Indicador de status de enriquecimento + fila de produtos que precisam de revisão.
- [ ] Override manual: editar nome/categoria/imagem; edição manual trava o campo contra sobrescrita do pipeline.
- [ ] Disparar re-enriquecimento de um produto sob demanda.
- [ ] Visão de execuções de sync ERP (status, contagens, erros) da S1.2/S1.4.

## Escopo / Fora de escopo
Inclui: telas admin de catálogo, enriquecimento e sync.
Não inclui: gestão de pedidos/operação (Fase 5).

## Notas técnicas
- Construído no shell admin (S0.7), consumindo a API de catálogo (S1.6) + endpoints admin.
- Campo travado por edição manual deve ser respeitado pelo pipeline da S1.5 (flag `locked`).
