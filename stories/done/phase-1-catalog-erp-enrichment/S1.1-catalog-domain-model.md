# S1.1 — Modelo de domínio de catálogo

- **Fase:** 1
- **Epic:** Catálogo
- **Status:** done
- **Depende de:** [S0.3]

## Objetivo
Modelo de dados de catálogo multi-loja: merchants, lojas, categorias, produtos, ofertas/preços e estoque.

## User story
Como plataforma, quero representar produtos de vários supermercados, para montar catálogo e carrinho multi-loja.

## Critérios de aceite
- [ ] Modelos Prisma: `Merchant`, `Store`, `Category`, `Product`, `Offer` (preço por loja), `Stock`.
- [ ] `Product` com identidade canônica (GTIN/EAN quando houver) + atributos (nome, marca, unidade, peso/volume, imagem).
- [ ] `Offer` liga `Product`↔`Store` com preço, preço promocional e disponibilidade.
- [ ] `Stock` por `Store`/`Product` (quantidade ou flag disponível).
- [ ] Categorias hierárquicas (Padaria, Hortifruti, Açougue, Bebidas, Mercearia — dos screenshots).
- [ ] Migration aplicada + índices (busca por nome, GTIN, store).
- [ ] Seed mínimo com 2 merchants e alguns produtos (ex.: Supermercado Europa, Condor).

## Escopo / Fora de escopo
Inclui: schema, relações, índices, seed.
Não inclui: ingestão via ERP (S1.2+), enriquecimento (S1.5), endpoints (S1.6).

## Notas técnicas
- Separar **produto canônico** (global) de **oferta** (por loja) é o que viabiliza dedup e enriquecimento entre lojas.
- Guardar `externalId` por origem (ERP) para reconciliar no sync.
- Tipos correspondentes em `packages/types`.
