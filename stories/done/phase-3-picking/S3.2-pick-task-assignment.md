# S3.2 Atribuição de tarefa ao separador
- **Fase:** 3
- **Epic:** Atribuição
- **Status:** done
- **Depende de:** [S3.1]

## Objetivo
Gerar tarefa de separação quando sub-pedido fica pago e atribuí-la a um separador da loja.

## User story
Como separador, quero ver e assumir pedidos da minha loja, para começar a separar.

## Critérios de aceite
- `OrderGroup` em `paid`/`preparing` gera `PickTask` com status `queued`
- Fila de tarefas por loja, ordenável (FIFO; agendados respeitam janela de S2.5)
- Separador assume tarefa (`queued → assigned`), vinculando `pickerId`
- Só `StoreStaff` com `staffRole=picker` da loja correspondente pode assumir (RBAC `picker` + escopo de `storeId`)
- Tarefa assumida não aparece para outros (lock); liberar volta para `queued`
- Evento emitido ao mudar de status (consumido por S3.8)

## Escopo / Fora de escopo
- **Fora:** separação item a item (S3.3)
- **Inclui:** geração, fila, atribuição manual e liberação

## Notas técnicas
- Geração via consumidor de evento de pedido pago (BullMQ) ou hook na transição
- Lock otimista no assume (evitar dois pickers na mesma tarefa)
- Atribuição automática (round-robin) fica fora; só manual nesta fase
