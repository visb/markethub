# S3.3 Sessão de separação item a item (API)
- **Fase:** 3
- **Epic:** Separação
- **Status:** done
- **Depende de:** [S3.2]

## Objetivo
API da sessão de separação: marcar cada item como separado, recusado ou a substituir (ref: `Picking.jpg`).

## User story
Como separador, quero percorrer os itens do pedido e registrar o que separei, para concluir a coleta.

## Critérios de aceite
- `POST /pick-tasks/:id/start` → `assigned → picking`
- `PATCH /pick-tasks/:id/items/:itemId` com ação: `pick | refuse | substitute`
- `pick` aceita `quantityPicked` (`weightGramsPicked` quando `saleType=weight`); valida contra quantidade pedida
- `refuse` exige motivo (ex: sem estoque, avariado)
- Não conclui sessão com itens `pending`
- `POST /pick-tasks/:id/complete-picking` → `picking → packed` (pré-condição: todos itens resolvidos)
- Idempotência por item; reanálise permitida antes de concluir
- Escopo: só o `pickerId` dono da tarefa edita

## Escopo / Fora de escopo
- **Fora:** fluxo de substituição/aprovação (S3.4), empacotamento (S3.5)
- **Inclui:** transições de item e da tarefa, validações

## Notas técnicas
- Recalcular subtotal do `OrderGroup` (e total do `Order`) conforme itens recusados/quantidade real
- Peso real pode diferir do pedido → ajuste de preço (cobrança/estorno tratado no pedido)
- Persistir autor e timestamp por ação (auditoria)
