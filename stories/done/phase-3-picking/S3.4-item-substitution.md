# S3.4 Substituição de item e ruptura de estoque
- **Fase:** 3
- **Epic:** Separação
- **Status:** done
- **Depende de:** [S3.3]

## Objetivo
Tratar item sem estoque: substituir por equivalente com aprovação do cliente, ou remover.

## User story
Como cliente, quero aprovar ou recusar um substituto quando faltar um item, para não receber algo que não quero.

## Critérios de aceite
- Separador propõe substituto (`Offer` da mesma loja) via ação `substitute`
- `Substitution` registra `OrderItem` original, `Offer` substituta, diferença de preço, `approvalStatus: pending`
- Cliente notificado (S3.8) e aprova/recusa; timeout → política padrão (ex: aceitar até X% mais barato, senão remover)
- Aprovado: item vira `substituted`, subtotal do `OrderGroup` ajustado
- Recusado: item vira `refused` (removido), valor estornado no acerto do pedido
- Diferença de preço nunca cobra acima do limite sem aprovação explícita

## Escopo / Fora de escopo
- **Fora:** motor de recomendação de substituto (manual nesta fase)
- **Inclui:** proposta, aprovação, ajuste de preço, política de timeout

## Notas técnicas
- Estorno/cobrança via provedor de pagamento (PIX/`Payment`) — reusa fluxo de S2.6
- Sugestão de substituto: busca simples por `Category`/marca entre `Offer`s da loja (sem ML)
- Aprovação assíncrona não deve travar a separação dos demais itens
