- **Fase:** 3

- **Status:** done
- **Depende de:** [S3.2, S3.3, S3.4, S3.5, S3.6]

> **Ajustado por SF.1:** removida a UI de empacotamento em caixas/QR/serial. Fluxo: separar → "Concluir separação" (`packed`) → "Pronto para coleta" (gera `pickupCode`) → tela de liberação com input do `pickupCode`.

## Objetivo
Implementar o app do separador (React Native/Expo) sobre o shell de S0.6.

## User story
Como separador, quero um app para ver minha fila, separar item a item e empacotar, para operar o dia todo no celular.

## Critérios de aceite
- Login `picker` e navegação por papel (shell de S0.6)
- Fila de tarefas da loja; assumir/liberar tarefa (S3.2)
- Tela de separação item a item: separar / recusar / substituir, com quantidade/peso (ref: `Picking.jpg`)
- Fluxo de substituição: propor substituto e ver status de aprovação (S3.4)
- Empacotamento: criar caixas, alocar itens, exibir QR/serial (ref: `Receive.jpg`)
- Botão "pronto para coleta" (S3.6)
- Atualizações em tempo real (S3.8); estados de loading/erro/offline básicos

## Escopo / Fora de escopo
- **Fora:** features de entregador e cliente
- **Inclui:** todas as telas de separação e empacotamento

## Notas técnicas
- Consumir API via `packages/api-client`; UI de `packages/ui`
- Sessão persistida (secure store)
- Entrada de peso para `saleType=weight` (g)
