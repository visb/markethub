# S2.2 — Endereços do cliente

- **Fase:** 2
- **Epic:** Marketplace (cliente)
- **Status:** done
- **Depende de:** [S2.1]

## Objetivo

CRUD de endereços do cliente, com endereço padrão, usado no checkout.

## User story

Como cliente, quero cadastrar e escolher meus endereços, para receber a entrega no lugar certo.

## Critérios de aceite

- [ ] Endpoints (role `customer`): `GET/POST/PATCH/DELETE /addresses`, `POST /addresses/:id/default`.
- [ ] Validação de CEP/UF; busca de CEP (ViaCEP) para autopreencher.
- [ ] Geocode lat/lng a partir do endereço (Google) para o motor de rotas (Fase 4) — pode ser stub agora.
- [ ] Um endereço padrão por cliente; trocar default é atômico.
- [ ] Tela no app cliente: listar/adicionar/editar/escolher endereço (ref: `Shipping Settings.jpg`).

## Escopo / Fora de escopo

Inclui: API de endereços + tela.
Não inclui: cálculo de frete por distância (S2.4/Fase 4).

## Notas técnicas

- Reusa auth/RBAC (`@Roles('customer')`, `@CurrentUser`).
- Contratos em `packages/types`; app usa `@markethub/api-client`.
- Geocode atrás de interface (trocável), igual ao padrão de providers da Fase 1.
