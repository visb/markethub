# S6.5 Favoritos de oferta (produto + loja)
- **Fase:** 6
- **Epic:** Refinamento cliente
- **Status:** todo
- **Depende de:** —

## Objetivo
"Salvar nos favoritos" do detalhe do produto vira funcionalidade real: guarda a oferta
(produto naquela loja) numa lista do usuário.

## User story
Como cliente, salvo as ofertas que compro sempre e as reencontro rápido em Minha Conta.

## Critérios de aceite
- [ ] Model `Favorite` (`userId` + `offerId`, unique composto, createdAt); referência fraca
      ao snapshot não é necessária — favorito de oferta indisponível aparece como
      "indisponível".
- [ ] API (auth customer): `POST /favorites { offerId }` (idempotente),
      `DELETE /favorites/:offerId`, `GET /favorites` → lista com produto, loja/mercado
      (nome+logo), preço atual e disponibilidade.
- [ ] Detalhe do produto: coração alterna salvar/remover (estado preenchido quando já é
      favorito) para a oferta principal exibida.
- [ ] Minha Conta: seção "Favoritos" com a lista; toque abre o detalhe do produto; botão
      adicionar ao carrinho direto da lista.
- [ ] Favorito duplicado não cria registro novo (upsert).

## Escopo / Fora de escopo
- Fora: favoritar loja/mercado (é o "Seguir" da loja — outra story); notificação de
  promoção em favorito.

## Notas técnicas
- `GET /favorites` resolve a Offer viva (preço/disponibilidade atuais) — sem snapshot de
  preço; se a Offer sumir, exibir produto como indisponível e permitir remover.
