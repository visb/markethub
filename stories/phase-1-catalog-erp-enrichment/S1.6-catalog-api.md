# S1.6 — API de catálogo

- **Fase:** 1
- **Epic:** Catálogo
- **Status:** done
- **Depende de:** [S1.1, S1.5]

## Objetivo
Endpoints públicos de catálogo que alimentam o app cliente: browse, busca e detalhe.

## User story
Como app cliente, quero consultar produtos por merchant, categoria e busca, para montar a vitrine e a página de produto.

## Critérios de aceite
- [ ] `GET /merchants` e `GET /merchants/:id/stores` (lojas disponíveis por região/endereço).
- [ ] `GET /stores/:id/categories` e listagem de produtos por categoria (paginada).
- [ ] `GET /search?q=` busca por produto, marca ou departamento (paginada, ranqueada).
- [ ] `GET /products/:id` detalhe com ofertas (preço por loja), disponibilidade e atributos enriquecidos.
- [ ] Seções de vitrine: ofertas em destaque, mais comprados, recomendados (regra simples no MVP).
- [ ] Respostas tipadas via `packages/types`; só produtos disponíveis aparecem por padrão.
- [ ] Paginação + filtros consistentes; performance com índices da S1.1.

## Escopo / Fora de escopo
Inclui: leitura de catálogo para o cliente.
Não inclui: carrinho/checkout (Fase 2); recomendação personalizada avançada (Fase 5).

## Notas técnicas
- Busca: começar com Postgres full-text/trigram; abstrair para trocar por engine dedicada depois.
- Cache de leitura (Redis) para vitrines quentes.
- Endpoints consumidos pelo `apps/customer` e validados pelo `apps/admin`.
