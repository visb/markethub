# S1.5 — Pipeline de enriquecimento (Cosmos Bluesoft + IA)

- **Fase:** 1
- **Epic:** Enriquecimento de dados
- **Status:** done (Cosmos real + Mock verificados; mapeamento heurístico — IA fica p/ depois)
- **Depende de:** [S1.1, S1.3]

## Objetivo
Enriquecer o catálogo via **Cosmos Bluesoft** (base de produtos por GTIN) e mapear as categorias do
Cosmos para as categorias da plataforma usando **IA**, melhorando nome, marca, imagem, NCM e categoria.

## User story
Como cliente, quero produtos com nome, imagem e categoria consistentes, para encontrar e comparar itens entre supermercados.

## Critérios de aceite
- [ ] Normalização de GTIN/EAN (validação de dígito verificador, padding 8/12/13/14).
- [ ] **Provider Cosmos**: lookup por GTIN em `https://cosmos.bluesoft.com.br/` (header `X-Cosmos-Token`),
      atrás de interface `EnrichmentProvider` (trocável/mockável). Respeitar rate limit + retry/backoff.
- [ ] **Cache de respostas Cosmos** (por GTIN) para não repetir chamada nem estourar cota.
- [ ] Enriquece `Product`: nome (description do Cosmos), marca, imagem (`thumbnail`), unidade/peso, GTIN,
      e guarda dados crus do Cosmos (NCM, GPC etc.) para auditoria.
- [ ] **Mapeamento de categorias por IA**: categoria do Cosmos → `Category` canônica da plataforma.
      O mapeamento é **persistido** (tabela `CategoryMapping`) e reusado; só chama a IA para categorias
      Cosmos ainda não mapeadas. Confiança baixa → marca para revisão manual.
- [ ] Dedup entre lojas por GTIN (já no sync) + reforço por nome+marca normalizados quando sem GTIN.
- [ ] Score de completude por produto (0-100) + `enrichmentStatus` (`pending`/`enriched`/`needs_review`).
- [ ] Pipeline roda como jobs BullMQ disparados após sync e re-executável sob demanda (por produto/loja).
- [ ] Idempotente e auditável (`provenance`: origem de cada campo enriquecido; campo travado por edição
      manual não é sobrescrito — flag `locked`).

## Escopo / Fora de escopo
Inclui: normalização, Cosmos provider + cache, mapeamento de categoria por IA, completude, provenance.
Não inclui: ML/embeddings próprio (Fase 5); revisão manual via admin (override básico na S1.7).

## Notas técnicas
- **Cosmos**: precisa de `COSMOS_TOKEN` (env). API retorna `description`, `brand`, `thumbnail`, `gtin`,
  `ncm`, `gpc`, `price` (referência), e `category`/`gpc` para mapear. Rate limit baixo → enfileirar +
  cache obrigatório. Provider isolado (`CosmosEnrichmentProvider implements EnrichmentProvider`) com um
  `MockEnrichmentProvider` para dev/test sem token.
- **IA p/ categorias**: usar Claude (Anthropic API, `ANTHROPIC_API_KEY`). Prompt recebe a categoria/GPC
  do Cosmos + a lista de `Category` da plataforma; responde a categoria-alvo + confiança. Resultado
  gravado em `CategoryMapping(cosmosCategory → categoryId, confidence, source)`. Cache evita recomputar.
- Modelos novos prováveis: `CategoryMapping`, `ProductEnrichment` (raw Cosmos + provenance) ou campos
  `Json` em `Product`. Decidir no design da story.
- Estratégia de match/normalização isolada e com alta cobertura de teste (coração da qualidade).
- Pendências a confirmar com usuário ao iniciar: token Cosmos disponível? rate limit/plano? modelo Claude
  a usar para o mapeamento?
