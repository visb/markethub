# S1.2 — Framework de conector ERP

- **Fase:** 1
- **Epic:** Integração ERP
- **Status:** done
- **Depende de:** [S1.1, S0.8]

## Objetivo
Camada abstrata (adapter pattern) que permite plugar qualquer ERP para sincronizar catálogo e empurrar pedidos.

## User story
Como plataforma, quero uma interface única de ERP, para integrar novos supermercados sem reescrever a lógica de sync.

## Critérios de aceite
- [ ] Interface `ErpConnector` com métodos: `fetchProducts`, `fetchPrices`, `fetchStock`, `pushOrder`, `acknowledge`.
- [ ] Modelo de normalização: DTO de origem → produto/oferta/estoque canônico.
- [ ] Registro de conectores (por merchant) + resolução por configuração.
- [ ] Orquestrador de sync agendável via BullMQ (jobs idempotentes, retry/backoff, dead-letter).
- [ ] Log de execução de sync (início, contagem, erros) persistido para auditoria.
- [ ] Mapeamento de `externalId`↔entidade canônica (upsert por origem).
- [ ] Testado com um conector fake em memória.

## Escopo / Fora de escopo
Inclui: contrato, orquestração, filas, reconciliação, auditoria.
Não inclui: conector real específico; conector mock concreto fica na S1.3.

## Notas técnicas
- Filas BullMQ + Redis (provisionados na S0.8).
- `pushOrder` fica stub até a Fase 2/3 gerarem pedidos.
- Desenhar para sync incremental (delta por timestamp/cursor) já aqui.
