# S2.3 — Carrinho multi-loja

- **Fase:** 2
- **Epic:** Marketplace (cliente)
- **Status:** done
- **Depende de:** [S2.1, S1.6]

## Objetivo
API de carrinho que suporta itens de várias lojas, com quantidades e variações de peso/porção.

## User story
Como cliente, quero juntar produtos de supermercados diferentes num carrinho, para comprar tudo de uma vez.

## Critérios de aceite
- [ ] Endpoints (role `customer`): `GET /cart`, `POST /cart/items`, `PATCH /cart/items/:id`, `DELETE /cart/items/:id`, `DELETE /cart`.
- [ ] Item referencia uma `Offer` (loja+produto); valida disponibilidade e preço atual.
- [ ] Input de quantidade dirigido por `Product.saleType` (ref: `Cart.jpg`):
  - `unit` (embalado) → contador de unidades (+/-).
  - `weight` (vendido por peso) → quantidade **em gramas** (ex.: 300g, 500g), nunca em kg.
- [ ] Carrinho agrupado por merchant, com subtotal por grupo e total geral.
- [ ] Item indisponível (estoque/sync) é sinalizado, não some silenciosamente.
- [ ] Resposta inclui contagem, subtotais e itens — pronta para a tela do carrinho.

## Escopo / Fora de escopo
Inclui: API de carrinho multi-loja + variações.
Não inclui: cupom/frete/taxas (S2.4), checkout (S2.5), telas (S2.8).

## Notas técnicas
- Preço do item = `Offer.promoPriceCents ?? priceCents` no momento; recalcular no checkout.
- `Product.saleType` (enum `unit|weight`) define a UI e o tipo de quantidade.
- Itens por peso: guardar `weightGrams` (inteiro, gramas); preço = `priceCents`(por kg) × gramas/1000.
- `Product.packageSize` é só rótulo descritivo ("2L","500g") para exibição.
- Reusa catálogo (S1.6) para validar oferta/disponibilidade.
