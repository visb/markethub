# S2.1 — Modelo de pedido, carrinho e endereço

- **Fase:** 2
- **Epic:** Marketplace (cliente)
- **Status:** todo
- **Depende de:** [S1.1]

## Objetivo
Modelar carrinho multi-loja, pedido, itens, endereço, cupom e taxas no schema Prisma.

## User story
Como plataforma, quero representar carrinho e pedido multi-loja, para suportar compra, pagamento e rastreio.

## Critérios de aceite
- [ ] `Address` (cliente): rótulo (Casa/Trabalho), rua/número/bairro/cidade/UF/CEP, complemento, lat/lng, `isDefault`.
- [ ] `Cart` (1 por cliente ativo) + `CartItem` (offer/produto, quantidade, variação de peso/porção, observação).
- [ ] `Order` + `OrderItem`: snapshot de preço no momento da compra, agrupado por merchant/loja.
- [ ] `OrderGroup` (sub-pedido por merchant): frete, taxa de preparo, taxa MarketHub, subtotal.
- [ ] `Coupon`: código, tipo (valor/percentual/frete grátis), validade, limites.
- [ ] Enums: `OrderStatus` (created, paid, preparing, picking, on_the_way, delivered, canceled), `PaymentStatus`.
- [ ] Migration aplicada + índices (cliente, status, criação).

## Escopo / Fora de escopo
Inclui: schema + migration de carrinho/pedido/endereço/cupom.
Não inclui: lógica de cálculo (S2.4), pagamento (S2.6), APIs (S2.2/2.3/2.5/2.7).

## Notas técnicas
- Preço sempre em centavos (Int), seguindo `Offer.priceCents`.
- `OrderItem` guarda snapshot (nome, preço, gtin) — catálogo pode mudar depois.
- Multi-loja: um `Order` agrupa vários `OrderGroup` (um por merchant), como nos screenshots
  (Europa #65654 + Condor #65464 num só pedido).
